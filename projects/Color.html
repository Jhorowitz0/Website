Overview

This project is a functional prototype 3D interface for selecting a color. This is the first project in in a series of case studies reexamining established 2D interfaces with 3D Interaction design principles in mind.

Exploring 3D...

Recently, the field of user interaction has begun to broaden to include 3D applications and interfaces. 3D UI design is a unique and young field with its own principals and languages. The goal of this project was to design an interface that fully utilizes three dimensionality instead of slapping a conventional 2D interface on a floating plane.

...But Why Color?

I decided to start with a color selection tool because color is a fundamentally 3D value; every color being composed of hue, saturation, and brightness. While the color picker as a UI has been explored ad nauseum in 2D, having the ability to express all 3 dimensions of color in a digestible, usable tool opens the interface up to new, 3D-focused forms of manipulation.

Making a Color

Color is a naturally three dimensional value. Every color is composed of 3 aspects:

Currently, most color picker interfaces take two of these values and map them to a 2D space, with the 3rd value having no choice but to be tacked on as a slider.

Having access to three dimensions allows each of these values (H,S,B) to be mapped directly to each spatial axis (XYZ). This creates a space containing every possible color. We'll refer to this as Color-Space going forwards.

Navigating Color-Space

Where to start?

While it’s fairly easy to map HSB to XYZ and create a cube of color, actually viewing and selecting values within that space is difficult, because we can only ever see 2D cross sections or the outside surface, never all the values within. The first step in this project was creating a way to easily view and navigate values within Color-Space.

Color-Space Divided

A natural starting point was to place a color point between 3 separate sliders. While this approach certainly creates a navigable interface, it's tricky to track 3 sliders at once. The user is manipulating three separate values, not one fluid color.

Plane & Pole

By combining saturation and brightness into a 2D plane along the hue slider, we better visualize various shades of each hue, allowing for easy navigation without overstimulation, splitting the difference between readability and usability.

Physicality in 3D Interfaces

A Design That Feels Present

When designing physical products, ergonomics are essential. the design needs to communicate which parts are interactable, and how those parts will move/change.

3D interface design is no different. Just because a design is no longer physical doesnt mean it shouldn't use physical design conventions to communicate function. Handles should seem grabbable, buttons should seem pressable in order to make an interface familiar.

For the plane and pole design shown above, making it appear mechanically accurate helps communicate how it will move before users interact with it. Proper physicality results in predictable and familiar 3D UI.

Old-School Lo-Fi

In order to quickly mock up an interface that feels present, the most simple solution was to actually create the interface physically. Similar to lo-fi mockups in Figma, This let me get quickly get a sense of scale, general feel, and usability.

Quick Changes

Usability tests found the interface clunky to circumnavigate, requiring a bit too much walking. As a result, an additional rotation gizmo was added.

Digital Application

Small design adjustments make the interface a lot more familiar - a stationary pole along the hue axis, caps to "stop" the color plane from falling off, a lazy susan inspired rotation gizmo for the design to rest and rotate on.

Touchable Holograms

Action and Reaction

The original 3D interfaces were physical objects. One essential feature of physical interfaces that 3D UI lacks is feedback. We’re acutely tuned to expect touch-based confirmation when we interact with the world around us. Turning handles activates a mechanism which we feel open a door. Pressing a button might give a satisfying pop.

Without the ability to properly communicate touch, 3D interfaces end up feeling “mushy”, and unresponsive. As a result, 3D interfaces need to compensate with a lot more finely tuned audiovisuals based on the state of the interface. Every action or even potential action should communicate confirmation to the user.

Visual Feedback

Having the color node grow on proximity and shrink on pinch results in a more responsive experience.

Audio

While audio might be neat flourish in a 2D interface, auditory response to actions is essential in making holograms have substance.

The Result

Final Tweaks

While the core interface is fluid and functional, the addition of a small pop-up menu allows for more in-depth information as well as fine tuning to specific color values. This project was a fun exploration into 3D thinking and hopefully it illustrates how exciting and unexplored the field of 3D UI design can be.
